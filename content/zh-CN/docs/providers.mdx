# 提供商

Neovate Code 支持多个提供商和模型。您可以使用自己的模型和提供商，或使用内置的模型和提供商。

## 内置提供商

<Callout type="info">
  没有找到您的提供商？提交问题或拉取请求来添加它。
</Callout>

### AIHubMix

1. 打开 [AIHubMix](https://aihubmix.com/)，创建账户并获取您的 API 密钥。

2. 输入 `/login` 并选择 **AIHubMix** 然后输入 API 密钥，或在您的环境变量 `AIHUBMIX_API_KEY` 中设置 API 密钥。

```bash
export AIHUBMIX_API_KEY=your-api-key
```

3. 运行 `/model` 来选择您想使用的模型。

### Antigravity

<Callout>
  基于 OAuth 认证，无需 API 密钥。
</Callout>

1. 输入 `/login` 并选择 **Antigravity**。

2. 按照 OAuth 流程使用您的 Google 账户进行认证。

3. 运行 `/model` 来选择您想使用的模型。

### Anthropic

1. 打开 [Anthropic Platform Console](https://console.anthropic.com/settings/keys)，点击 **Create Key** 创建新的 API 密钥并复制它。

2. 输入 `/login` 并选择 **Anthropic** 然后输入 API 密钥，或在您的环境变量 `ANTHROPIC_API_KEY` 中设置 API 密钥。

```bash
export ANTHROPIC_API_KEY=your-api-key
```

> 如果您想使用自定义基础 URL，可以设置环境变量 `ANTHROPIC_API_BASE`。

```bash
export ANTHROPIC_API_BASE=https://your-base-url
```

3. 运行 `/model` 来选择您想使用的模型。

## 自定义提供商

您可以添加任何未内置的兼容 OpenAI 的提供商。

编辑 `~/.neovate/config.json` 文件来添加提供商。

```json
{
  "provider": {
    "your-provider": {
      "apiKey": "your-api-key",
      "api": "https://your-base-url",
      "doc": "https://your-doc", // 可选
      "env": ["YOUR_API_KEY"], // 可选
      "apiEnv": ["YOUR_API_BASE"], // 可选
      "name": "your-provider", // 可选
      "models": {
        // 值为模型 id
        // 完整列表可在 https://github.com/neovateai/neovate-code/blob/d1b8cc5/src/model.ts#L70 找到
        "foo": "claude-4-sonnet"
      }
    }
  }
}
```

<Callout type="info">
  想要更多控制权？您可以使用[插件](/docs/plugins)创建自己的代码代理，通过 `provider` 和 `modelAlias` 钩子获得对提供商配置、模型别名等的完全编程控制。
</Callout>
